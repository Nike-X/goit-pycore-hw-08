# Assistant Bot Address Book (`goit-pycore-hw-08`)

Console assistant for managing contacts, phone numbers, and birthdays.

> Disclaimer: This `README.md` file was generated by OpenAI Codex agent.

The project contains:
- `main.py` - CLI bot, command parser, command handlers, and user-facing messages.
- `adress_book.py` - core data model (`Field`, `Name`, `Phone`, `Birthday`, `Record`, `AddressBook`).

## Features

- Add new contacts.
- Add multiple phone numbers to one contact.
- Change an existing phone number.
- Show all numbers for a contact.
- Add and show birthday for a contact.
- Show all contacts.
- Show upcoming birthdays for the next 7 days.
- Weekend birthday handling:
  - Saturday and Sunday congratulations are moved to Monday.

## Data Model

- `Field` - base class for value fields.
- `Name` - validates non-empty contact name.
- `Phone` - validates phone format: exactly 10 digits.
- `Birthday` - validates date format: `DD.MM.YYYY`.
- `Record` - one contact: name, list of phones, optional birthday.
- `AddressBook` - container for records (`UserDict`) with search/delete and upcoming birthdays logic.

## Requirements

- Python 3.8+ (no external dependencies).

## Run

```bash
python main.py
```

## Commands

| Command | Description | Example |
|---|---|---|
| `hello` | Greeting | `hello` |
| `add <name> <phone>` | Add contact or add phone to existing contact | `add John 1234567890` |
| `change <name> <old_phone> <new_phone>` | Replace one phone with another | `change John 1234567890 0987654321` |
| `phone <name>` | Show all phones for contact | `phone John` |
| `all` | Show all contacts (with birthday if set) | `all` |
| `add-birthday <name> <dd.mm.yyyy>` | Set birthday for contact | `add-birthday John 10.03.1990` |
| `show-birthday <name>` | Show contact birthday | `show-birthday John` |
| `birthdays` | Show contacts with birthdays in next 7 days | `birthdays` |
| `close` / `exit` | Exit bot | `exit` |

## Validation and Error Handling

- Invalid command arguments are wrapped as: `Invalid arguments: <details>`.
- Invalid phone format details: `Invalid phone number: <value>. Phone number must contain exactly 10 digits.`
- Invalid birthday format details: `Invalid date format: <value>. Use DD.MM.YYYY`
- Missing command arguments returns: `Not enough arguments for this command.`
- Unknown contact returns: `Requested contact does not exist.`
- Any other unexpected exception is returned as: `An unexpected error occured: <Type> <message>`
- Unknown command returns: `Invalid command.`

## Notes

- File name is `adress_book.py` (as in repository), and imports in `main.py` depend on this exact name.
- Address book data is persisted with `pickle` in `addressbook.pkl`:
  - loaded on startup (or empty book is created if loading fails),
  - saved automatically when bot exits with `close` or `exit`.
- `birthdays` includes contacts where next birthday is in the range `today <= birthday < today + 7 days` (today included, day 7 excluded).
